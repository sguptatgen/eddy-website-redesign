EDDY: Evaluation of Differential DependencY

This page provides a Java implementation of an algorithm EDDY, which was presented in "EDDY: A novel statistical gene set test method to detect differential genetic dependencies".

What is EDDY?

Identifying differential features between conditions is a popular approach to understanding molecular features and their mechanisms underlying a biological process of particular interest. Although many tests for identifying differential expression of gene or gene sets have been proposed, there was limited success in developing methods for differential interactions of genes between conditions because of its computational complexity. We present a method for Evaluation of Dependency DifferentialitY (EDDY), which is a statistical test for differential dependencies of a set of genes between two conditions. Unlike previous methods focused on differential expression of individual genes or correlation changes of individual gene-gene interactions, EDDY compares two conditions by evaluating the probability distributions of dependency networks from genes. The method has been evaluated and compared with other methods through simulation studies, and application to glioblastoma multiforme data resulted in informative cancer and glioblastoma multiforme subtype-related findings. The comparison with Gene Set Enrichment Analysis, a differential expression-based method, revealed that EDDY identifies the gene sets that are complementary to those identified by Gene Set Enrichment Analysis. EDDY also showed much lower false positives than Gene Set Co-expression Analysis, a method based on correlation changes of individual gene-gene interactions, thus providing more informative results. The Java implementation of the algorithm is freely available to noncommercial users.

Downloads

Command Line EDDY

RunEDDY.jar is a runnable JAR file that can be used to run a single instance of EDDY from the command line.

Download RunEDDY.jar

EDDY API

Eddy.jar is an API version of the EDDY algorithm implementation.

Download eddy.jar

Flanagan Library

Michael Thomas Flanagan's Java Scientific Library, Flanagan.jar, is a required library for running EDDY. More information on the Flanagan library can be found at http://www.ee.ucl.ac.uk/~mflanaga.

Download flanagan.jar

System Requirements

Programs were implemented and tested on Java version 1.6.0. Requires Java virtual machine 1.6.0 or higher.

File Formats

Input data file

The input data file is a tab-delimited text file. Each row is a variable and each column is a sample. It needs both of row and column headers for variable names and sample names. Variable values should be categorial. The following file is a sample input data file:

TCGA-GBM.txt
Glioblastoma gene expression data from The Cancer Genome Atlas (TCGA). Refer the publication for preprocessing information.

Sample class information file

The sample class information file is a tab-delimited single line text file. Two-class information needs to be listed in the order of samples from the input data file. The following files are examples of sample class information files:

Mesenchymal.txt
Sample class information with Mesenchymal subtype versus the other subtypes.
Proneural.txt
Sample class information with Proneural subtype versus the other subtypes.

Gene set file (for test)

The gene set file is a tab-delimited single line text file with gene names (or variable names for general applications). The following files are sample gene set files:

ATP_DEPENDENT_DNA_HELICASE_ACTIVITY.txt
BIOCARTA_FEEDER_PATHWAY.txt
BIOCARTA_TALL1_PATHWAY.txt
ST_JAK_STAT_PATHWAY.txt

Operating the RunEDDY.jar Command Line Program

For command line options, execute the following command:

% java -jar RunEDDY.jar

You may use the following options with the command:

-d input data file
-t target variable list file
-c class information file
-pp lambda parameter
-m maximum number of allowed incoming edges for each variable, when variable dependency networks are enumerated. -1 indicates no limitation.
-s "all" or the number of enumerated dependency network structures for EDDY (M parameter)
-pm number of permutations

The following is an example command:

% java -jar RunEDDY.jar -d TCGA-GBM.txt -t ATP_DEPENDENT_DNA_HELICASE_ACTIVITY.txt -c Proneural.txt -pp 1 -m 3 -s 1000 -pm 1000

In case of low memory error (java.lang.OutOfMemoryError Exception, from Java heap space), use -Xmx option for Java virtual machine to increase the amount of memory. For example:

% java -Xmx512m -jar RunEDDY.jar
(Use up to 512MB heap space for this execution of RunEDDY.jar.)

Operating the EDDY API

Use eddy.jar library for your own Java program.

The necessary classes for eddy.jar are eddy.Data and eddy.EDDY

Use "Data" class to read an input data file. For example:

String fileName = "TCGA-GBM.txt";
Data data = new Data(fileName);

An instance of EDDY class can be created with the following EDDY constructor:

public EDDY(Data inputData, Vector<String> targetGene, String sampleClassInformationFileName, int amountOfPermutation, boolean isConsideringAllStructure, int numberOfProbableStructure, double lambda, int maxParentAmount, int timeLimit, boolean isSilent)
Data inputData A Data instance of an input data file
Vector<String> targetGene A Vector of String with the name of target genes for test
String sampleClassInformationFileName Sample class information file
int amountOfPermutation The number of permutations
boolean isConsideringAllStructure If true, "numberOfProbableStructure" is ignored and all possible DAG structures are enumerated. If false, "numberOfProbableStructure" DAG structures are enumerated.
int numberOfProbableStructure M value of EDDY
double lambda Lambda parameter of EDDY
int maxParentAmount The number of maximum allowed incoming edges in enumerating DAG structures.
int timeLimit Stop permutation after "timeLimit" hours of running and returns result.
boolean isSilent If true, no screen message is printed.

Create an EDDY instance, and run test:

EDDY ed = new EDDY(...);
double pValue = ed.runTest();

Additional information can be accessed from the members of EDDY class after test:

EDDY.JS JS discrepancy from the test
EDDY.pValue p-value from the test
EDDY.permutedAmount The number of permutations done within the given time limit
